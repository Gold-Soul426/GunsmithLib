# 多弹种功能
### 概念：
在介绍配置方法之前，您可能需要了解下面几个概念：
* 这个文件的作用本质上是允许玩家可以在指定的枪械 id 列表中自由切换。
* 武器变体集合 (variant set) 是一个数据包文件，里面存放了一把武器所有的部位/变体/开火模式变体的数据。
* 武器部位：通常代表一个武器不同的部分，例如步枪的发射机和下挂榴弹就是两个不同的部位。武器会保留每个部位的弹药量W，开火模式，选择的变体。
* 武器变体：代表一把武器对应的多个弹种/模式。
* 武器开火模式变体：代表一个武器变体在不同的开火模式下使用的枪械 id。
### 武器变体集合文件的存放位置：
`data/<枪包的命名空间>/gunsmithlib/gun_ammo_variant_sets/<任意有效资源名称>.json`
### 武器变体集合文件的内容
```json5
{
  // 弹种武器的本体，
  // 决定系统会使用哪个 model 和 animation。
  "master": "tacz:mk14",
  // 弹种组的优先级。影响实际生效的 master 的值与合并后的 parts 的顺序。
  // 警告：多个弹种组使用同一个 priority 将导致游戏崩溃。
  //
  // 下面的特殊值是为自带弹种支持的枪包准备的。
  // 为他人的枪包添加弹种支持时强烈建议不要使用下面的特殊值。
  //
  // 特殊值：
  // - 如果 priority 是一个有效的 ResourceLocation 格式，
  //   那么 namespace 按 "tacz" < "std" < 其他值排序，
  //   同时 path 按 "master" < "main" < 其他值排序。
  //
  // - 如果不符合 ResourceLocation 的要求，
  //   那么特殊优先级按 "master" < "main" < "tacz" < "std" < 其他值的顺序排序。
  //
  // 6.0.0 新增
  "priority": "std:main",
  // "parts" 键名称固定，值是武器的不同部位。
  // 如果你不需要多部位功能，可以只填一个部位，部位名称随意。
  "parts": [
    {
      // 部位名称，值可以是任何字符串。
      "name": "main",
      // 部件的优先级。影响长按 X 键打开的转盘里的排序。
      // 警告：多个弹种组的同名部件使用同一个 priority 将导致游戏崩溃。
      // 特殊值同上面那个 priority。
      // 6.0.0 新增
      "priority": "std:main",
      // 当前部位的变体列表
      "variants": [
        // 字符串意味着当前变体的枪械 id 与开火模式无关。
        "tacz:mk14",
        // json 对象意味着当前变体的枪械 id 与开火模式有关。
        // 对象的键代表开火模式，开火模式的值可以为 "auto", "semi", "burst"。
        {
          "auto": "gunsmithlib:mk14_ap",
          "semi": "gunsmithlib:mk14_ap_semi"
        }
      ]
    }
  ]
}
```
### 脚本编写
请参照 [scripting.md](scripting.md)，特别注意版本 `4.12.0` 中新增的变量。
### 注意事项
* 同一武器变体集合中的所有武器将会使用同一个模型和状态机。
